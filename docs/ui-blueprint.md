# Trading Workspace UI Blueprint

## Purpose

This document defines the long-term UI architecture of Trading Workspace.

The objective is to create a professional, modular, keyboard-driven trading workspace rather than a traditional trading website.

---

# Design Principles

## 1. Chart First

The chart is always the primary focus.

UI should never permanently block important chart information.

---

## 2. Floating Workspace

Secondary tools should appear only when needed.

Examples:

* Order Entry
* Replay
* Alerts
* Heatmap
* AI Assistant

These panels should be easy to open and close.

---

## 3. Keyboard First

Every important feature should be accessible by keyboard.

Mouse interaction should remain optional.

---

## 4. Modular Design

Every functional area should become an independent module.

Examples:

* Chart Module
* Order Module
* Replay Module
* Position Module
* Heatmap Module
* Alert Module

---

# Planned Workspace Layout

## Main Area

* Chart
* Indicators
* Drawing Tools

---

## Right Side

Dockable Panels

* Order Entry
* Open Positions
* Watchlist

---

## Floating Panels

Temporary windows that appear on demand.

Examples:

* Replay Control
* Leverage Settings
* Position Size
* AI Assistant

---

## Notification System

Support non-blocking notifications.

Examples:

* Order Opened
* Order Closed
* TP Triggered
* SL Triggered
* Replay Finished

Notifications should disappear automatically.

---

## Task Center

A future task panel for:

* Development tasks
* Trading checklists
* Replay notes
